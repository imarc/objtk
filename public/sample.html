Headings

There are two types of headings supported in ObjTK, however, they might not do what you expect. Read on to find out more.

  • Classic Headings
  • Structural Headings

Classic Headings

Classic headings use the h1 through h6 tags. Historically, these were used almost exclusively to determine the style of the heading. With the advent of HTML5 and the introduction of the section tag, it is now possible to actually structure a document semantically. As such, we suggest you use and, by default, style h1 through h6 differently than what you might be used to.

Instead of using the h2 as simply a different style, use it to follow an h1 within the same parent element, indicating it as a secondary title for the same section (not a primary title for a secondary section).

Block Content

Block content includes things like paragraphs, lists, block quotes, code blocks, etc. This section will explain the default styling on some of these and why we made the decisions we did. Block content will only have bottom margin except in the case of some specific nesting behavior such a lists.

The mixin uses the ::before pseudo-element in order to display and control bullet content which is a fairly common practice. However, because this often breaks expectations, particularly when someone is trying to figure out why they can't get bullets to disappear with list-style, we explicitly only style non-classed lists in this manner. If you want other lists with classes to be styled as such, we suggest including the mixin.

A few of our goals when creating lists include:

  • Ensuring consistent bullet sizes and meticulous placement
